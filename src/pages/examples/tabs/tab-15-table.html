Table Component

A sortable, exportable data table that accepts JSON attributes set directly in HTML for static data, or JavaScript property setters for dynamic content.

Static Data

Columns and rows defined directly via JSON attributes. Sortable columns are marked with sortable: true. Zebra striping and CSV export are both enabled.

Dynamic Data

Table columns and rows populated at runtime via JavaScript property setters. Click the button to fetch real users from the JSONPlaceholder API.

Search

Add the search attribute to render a live search input above the table. Typing filters rows across all columns (case-insensitive). Combine with csv-export and zebra as needed.

Sticky Header & Scrollable Body

Add sticky-header to pin the header row and max-rows="N" to limit the visible row count and enable vertical scrolling. The two attributes are independent: you can use either or both.

Fetch Users

Editable Data

The Name column renders a td-text-input per row. Edits dispatch a cell-change event on the table element: useful for inline editing workflows.

Custom HTML Table

Place <thead> and <tbody> children directly inside <td-table> to author the table manually. All component styles: borders, zebra striping, hover states: still apply.

Nested / Grouped Column Headers

Add a children array to any column definition to create a spanning group header. Nesting is unlimited. Leaf columns (those without children) control the data keys and can still be sortable. Works with sticky-header too; each header row sticks independently.

Highlighting

Rows and individual cells can be coloured using the same four variants as the button component. Use a badge span inside a render() function for inline text labels.

Attributes

  • columns: JSON array of { key, label, sortable? } objects
  • data: JSON array of row objects keyed to column keys
  • search: Boolean: shows a live search input that filters rows across all columns
  • csv-export: Boolean: shows the Export CSV button above the table
  • csv-filename: Download filename without extension (default: "export")
  • zebra: Boolean: applies alternating row background colours (works in all modes)

Nested Headers Usage

Add a children array to any column to create a spanning group header row. Group headers have no key and cannot be sorted. Leaf columns (those without children) drive the data and can be sortable. Nesting can go as deep as needed.

Column fields

  • key: Data property name. Use "" for group headers that have no direct data column
  • label: Header text
  • sortable: Boolean: enables click/keyboard sort on leaf columns only
  • children: Array of sub-columns. Presence of this field makes the column a group header; key and sortable are ignored
  • render: Optional function (value, row) => string: return an HTML string to fully control the cell content
