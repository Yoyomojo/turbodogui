Alert Component

Display contextual feedback messages with four semantic variants: info, success, warning, and alert. Alerts can be used inline as static banners or as toast notifications - fixed overlay messages that stack in a corner of the viewport, slide in with an animation, and optionally auto-dismiss after a configurable delay.

Variants

Four semantic variants communicate intent at a glance:

This is an informational message. Your changes have been saved successfully. Your session will expire in 5 minutes. An error occurred while processing your request.

With Title

Use the title attribute to add a bold heading above the message:

You can use the title attribute to add a heading. All 3 files were processed without errors. You are using 90% of your available storage. Please verify your email address to continue.

Dismissible

Add the dismissible attribute to show a close button. Clicking it hides the alert and emits a dismiss event:

Click the Ã - to dismiss this alert. Your plan has been upgraded to Pro. You have unsaved changes that will be lost.

Toast Notifications

Add the toast attribute to display the alert as a fixed toast notification. Use toast-position to choose the corner and duration (ms) for auto-dismiss. Toast alerts always include a close button.

Component Features

  • Variants: info, success, warning, alert
  • Title: Optional bold heading via the title attribute
  • Dismissible: Boolean attribute - shows a close button that hides the alert
  • Toast: Boolean attribute - renders as a fixed overlay toast with a built-in close button; always closable in toast mode
  • Toast Position: toast-position - top-right (default), top-left, bottom-right, bottom-left, top-center, bottom-center
  • Duration: duration (ms) - auto-dismiss delay; omit or 0 for no auto-dismiss
  • Slot: Default slot accepts any HTML content
  • Theme: Colors automatically adapt to light and dark mode
  • Events: Emits dismiss when closed by button or auto-dismiss
