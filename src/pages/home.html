Core

  • Web Components - every UI element is a native Custom Element with Shadow DOM encapsulation, built in TypeScript
  • Router - lightweight pathname-based router with dynamic route params, lifecycle listeners, and SEO-friendly title/description support
  • Theme system - light/dark mode via CSS custom properties with localStorage persistence and system-preference detection
  • HTTP helper - thin fetch wrapper with automatic JSON serialisation/parsing, Content-Type injection, and non-2xx error throwing
  • Responsive grid - mobile-first 12-column layout with sm (576 px), md (768 px), lg (992 px), and xl (1200 px) breakpoints
  • Zero runtime dependencies - dev tooling only: TypeScript ^5.6.3 and Vite ^8.0.8

App Shell

The app uses a shared shell so navigation, theming, and layout stay consistent across all routes.

Structure

  • Top bar in src/shared/app-shell.html provides branding, Download from Github link, and theme toggle.
  • Sidebar (<td-sidebar>) contains component and layout navigation entries.
  • Main outlet (#app-content) is where route HTML is rendered.

Runtime Flow

Each route returns page markup, while the shell remains mounted once at startup.

Why This Pattern

  • Consistency: one navigation and layout frame for every page.
  • Simplicity: routes only define page content, not shared chrome.
  • Scalability: adding pages means registering a route and linking it in sidebar/nav.

Use It As A Starter

If you want a fresh app, keep the shell and the pieces you need, then remove the demo content you do not. This repository is already wired so you can strip it down fast and build on top of it without touching the core plumbing.

Suggested cleanup

  • Delete public/example-video/ if you do not need the sample media assets.
  • Remove example pages you are not using from src/pages/examples/.
  • Trim the component list in src/pages/home.html and the sidebar nav if you want a smaller app.
  • Keep src/core/, src/shared/, and the components you plan to reuse.

How to build on it quickly

  • Create a new HTML page in src/pages/.
  • Register the route in src/pages/app-routes.ts.
  • Build new UI with existing components, or add a component under src/components/.
  • Fetch data with the shared http helper and bind it into your route or component.

API Calls

The http helper in src/core/http.ts provides a lightweight fetch wrapper with built-in JSON handling and error handling.

Create an API Module

Create src/api/users-api.ts:

import { http } from "../core/http"; export interface User { id: number; name: string; email: string; } export const getUsers = () => http<User[]>("/api/users"); export const getUser = (id: number) => http<User>(`/api/users/${id}`); export const createUser = (data: Omit<User, "id">) => http<User>("/api/users", { method: "POST", body: data });

Using the HTTP Helper

const data = await http("/api/example"); const users = await http<User[]>("/api/users"); const result = await http("/api/data", { method: "POST", headers: { "Authorization": "Bearer token" }, body: { name: "John" } });

Populating Pages with Data

Since pages are rendered server-side in the route handler, fetch data before rendering the HTML:

Fetch and Render Pattern

router.register("/users", async () => { try { const users = await getUsers(); const usersList = users .map(u => `<li>${u.name} - ${u.email}</li>`) .join(""); return navHtml + ` <div class="container"> <ul>${usersList}</ul> </div> `; } catch (error) { return navHtml + ` <div class="container"> <p>Error loading users</p> </div> `; } }, { title: "Users", description: "List of all users" });

Using Web Components for Dynamic Content

For more complex, interactive pages, use Web Components:

import { UIComponent } from "../core/component"; import { getUsers } from "../api/users-api"; export class UsersPage extends UIComponent { protected async render(): Promise<void> { try { const users = await getUsers(); this.root.innerHTML = ` <ul> ${users.map(u => `<li>${u.name}</li>`).join("")} </ul> `; } catch (error) { this.root.innerHTML = `<p>Error loading users</p>`; } } } customElements.define("users-page", UsersPage);

Creating Web Components

Extend the UIComponent base class to create reusable components. Place each component in its own folder with a paired .css file - Vite's ?raw import brings it in as a string at build time so it lands inside the Shadow DOM.

File structure

src/components/my-component/ my-component.ts my-component.css

my-component.css

:host { display: block; } .my-component { padding: 1rem; background: var(--td-surface); color: var(--td-fg); border-radius: var(--td-radius); }

my-component.ts

import { UIComponent } from "../../core/component"; import styles from "./my-component.css?raw"; export class MyComponent extends UIComponent { static tag = "my-component"; static observedAttributes = ["label", "disabled"]; attributeChangedCallback() { this.render(); } protected render(): void { const label = this.getAttribute("label") ?? "Default"; const disabled = this.hasAttribute("disabled"); this.root.innerHTML = ` ${this.css(styles)} <div class="my-component"> <button ${disabled ? "disabled" : ""}>${label}</button> <slot></slot> </div> `; } } customElements.define(MyComponent.tag, MyComponent);

Register in src/index.ts

import "./components/my-component/my-component";

Use in any page HTML

Key points

  • ?raw - Vite import suffix that returns the file contents as a plain string
  • this.css(styles) - wraps the string in a <style> tag scoped inside the Shadow DOM
  • :host - selects the custom element itself from within its own stylesheet
  • static observedAttributes - list attributes that should trigger attributeChangedCallback and re-render
  • <slot> - projects child content from the light DOM into the Shadow DOM

Environment Variables

Vite loads .env.development during npm run dev and .env.production during npm run build. Copy the example templates to get started:

cp .env.development.example .env.development cp .env.production.example .env.production

All variables exposed to the client must be prefixed with VITE_:

.env.development

VITE_APP_NAME=Turbodog UI VITE_API_URL=http://localhost:3000 VITE_APP_ENV=development

.env.production

VITE_APP_NAME=Turbodog UI VITE_API_URL=https://api.example.com VITE_APP_ENV=production

Usage

const appName = import.meta.env.VITE_APP_NAME; const apiUrl = import.meta.env.VITE_API_URL; const env = import.meta.env.VITE_APP_ENV;

Add new variables to src/env.d.ts to keep them typed.

Theming

The application uses CSS variables for theming. Colors are defined in src/theme/base-theme.css and can be toggled between light and dark modes.

import { initializeTheme, toggleTheme } from "./src/theme/theme"; initializeTheme(); // Initialize on app start toggleTheme(); // Toggle between light/dark

The selected theme is persisted in localStorage under the key turbodogtd-theme.
