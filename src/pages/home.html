Turbodog UI

A lightweight TypeScript Web Component library with a tiny router, theme system, shared fetch helper, and responsive grid.

The grid starts mobile-first and expands through sm, md, lg, and xl breakpoints.

Getting Started

Clone the repository and run the following commands:

npm install
npm run dev

Grid System

12-column layout system with responsive breakpoints:

  • sm: 576px+
  • md: 768px+
  • lg: 992px+
  • xl: 1200px+

Adding Routes

Routes are registered in src/main.ts using the router. Each route maps a path to a page and optional metadata.

Register a Route

import { router } from "../core/router";
import navHtml from "../shared/nav.html?raw";
import myPageHtml from "./my-page.html?raw";

router.register("/my-page", () => navHtml + myPageHtml, {
  title: "My Page",
  description: "Description for SEO"
});

Dynamic Route Parameters

Routes support dynamic parameters using colon syntax :param:

router.register("/users/:id", (params) => {
  return navHtml + `<h1>User ${params.id}</h1>`;
}, {
  title: "User Details"
});

Navigate to routes using the router.navigate() method or the <ui-link> component.

Creating Pages

Pages are HTML files stored in src/pages/ and imported as raw strings in main.ts.

Create a Page File

Create src/pages/my-page.html:

<div class="container app-shell">
  <div class="row row--center">
    <div class="col-12 col-lg-12">
      <ui-card>
        <h2>My Page</h2>
        <p>Page content goes here</p>
      </ui-card>
    </div>
  </div>
</div>

Import and Register

In src/main.ts, import the page and register it:

import myPageHtml from "./my-page.html?raw";

router.register("/my-page", () => navHtml + myPageHtml);

API Calls

The http helper in src/api/http.ts provides a lightweight fetch wrapper with built-in JSON handling and error handling.

Create an API Module

Create src/api/users-api.ts:

import { http } from "./http";

export interface User {
  id: number;
  name: string;
  email: string;
}

export const getUsers = () => http<User[]>("/api/users");

export const getUser = (id: number) => http<User>(`/api/users/${id}`);

export const createUser = (data: Omit<User, "id">) =>
  http<User>("/api/users", {
    method: "POST",
    body: data
  });

Using the HTTP Helper

const data = await http("/api/example");

// With type safety
const users = await http<User[]>("/api/users");

// With request options
const result = await http("/api/data", {
  method: "POST",
  headers: { "Authorization": "Bearer token" },
  body: { name: "John" }
});

Populating Pages with Data

Since pages are rendered server-side in the route handler, fetch data before rendering the HTML:

Fetch and Render Pattern

// In main.ts
router.register("/users", async () => {
  try {
    const users = await getUsers();
    const usersList = users
      .map(u => `<li>${u.name} - ${u.email}</li>`)
      .join("");

    return navHtml + `
      <div class="container">
        <ul>${usersList}</ul>
      </div>
    `;
  } catch (error) {
    return navHtml + `
      <div class="container">
        <p>Error loading users</p>
      </div>
    `;
  }
}, {
  title: "Users",
  description: "List of all users"
});

Using Web Components for Dynamic Content

For more complex, interactive pages, use Web Components:

// src/pages/users-page.ts
import { UIComponent } from "../core/component";
import { getUsers } from "../api/users-api";

export class UsersPage extends UIComponent {
  protected async render(): Promise<void> {
    try {
      const users = await getUsers();
      this.root.innerHTML = `
        <ul>
          ${users.map(u => `<li>${u.name}</li>`).join("")}
        </ul>
      `;
    } catch (error) {
      this.root.innerHTML = `<p>Error loading users</p>`;
    }
  }
}

customElements.define("users-page", UsersPage);

Creating Web Components

Extend the UIComponent base class to create reusable components. Place each component in its own folder with a paired .css file — Vite's ?raw import brings it in as a string at build time so it lands inside the Shadow DOM.

File structure

src/components/my-component/
  my-component.ts
  my-component.css

my-component.css

:host {
  display: block;
}

.my-component {
  padding: 1rem;
  background: var(--ui-surface);
  color: var(--ui-fg);
  border-radius: var(--ui-radius);
}

my-component.ts

import { UIComponent } from "../../core/component";
import styles from "./my-component.css?raw";

export class MyComponent extends UIComponent {
  static tag = "my-component";
  static observedAttributes = ["label", "disabled"];

  attributeChangedCallback() {
    this.render();
  }

  protected render(): void {
    const label = this.getAttribute("label") ?? "Default";
    const disabled = this.hasAttribute("disabled");

    this.root.innerHTML = `
      ${this.css(styles)}
      <div class="my-component">
        <button ${disabled ? "disabled" : ""}>${label}</button>
        <slot></slot>
      </div>
    `;
  }
}

customElements.define(MyComponent.tag, MyComponent);

Register in src/index.ts

import "./components/my-component/my-component";

Use in any page HTML

<my-component label="Click me">
  <p>Slotted content</p>
</my-component>

Key points

  • ?raw — Vite import suffix that returns the file contents as a plain string
  • this.css(styles) — wraps the string in a <style> tag scoped inside the Shadow DOM
  • :host — selects the custom element itself from within its own stylesheet
  • static observedAttributes — list attributes that should trigger attributeChangedCallback and re-render
  • <slot> — projects child content from the light DOM into the Shadow DOM

Environment Variables

Vite loads .env.development during npm run dev and .env.production during npm run build. Copy the example templates to get started:

cp .env.development.example .env.development
cp .env.production.example .env.production

All variables exposed to the client must be prefixed with VITE_:

.env.development

VITE_API_URL=http://localhost:3000
VITE_APP_ENV=development

.env.production

VITE_API_URL=https://api.example.com
VITE_APP_ENV=production

Usage

const apiUrl = import.meta.env.VITE_API_URL;
const env = import.meta.env.VITE_APP_ENV;

Add new variables to src/env.d.ts to keep them typed.

Theming

The application uses CSS variables for theming. Colors are defined in src/theme/base-theme.css and can be toggled between light and dark modes.

import { initializeTheme, toggleTheme } from "./src/theme/theme";

initializeTheme(); // Initialize on app start
toggleTheme(); // Toggle between light/dark

The selected theme is persisted in localStorage under the key turbodogui-theme.
